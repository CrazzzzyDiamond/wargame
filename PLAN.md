# Wargame — План розробки

## Відкриті питання

- [ ] **Бойовий контакт** — як він вирішується? Автоматичний розрахунок з рандомом, чи гравець щось вирішує в момент зіткнення?
- [ ] **Дезінформаційна механіка** — гравець натискає кнопку і витрачає ресурс, чи тонше — через реальні переміщення хибних юнітів?
- [ ] **Туман війни — візуалізація** — як гравець бачить "застарілу" розвідку? Прозорість? Поступове зникнення позицій?
- [ ] **MVP scope** — що з 7 пунктів найризикованіший технічно?
- [x] **Режим гри** — реальний час з паузою і контролем швидкості

### Юніти гравця
- [x] Ієрархія: Бригада (HQ) → Батальйон (HQ) → Роти (пряме керування гравця)
- [x] Різноманіття рот дає тактичну глибину: штурмові, танкові, розвідувальні, протитанкові, мінометні, інженерні
- [x] Спецура (Кракен, ССО) — окремі групи, швидкі, невидимі у тумані війни
- [x] ТРО — утримання позицій, слабкі в атаці
- [ ] Скільки рот від кожного типу бригади? (визначає загальну кількість юнітів)
- [ ] Туман війни — видимість на рівні рот чи батальйонів?
- [ ] Підкріплення по ходу гри чи всі юніти з початку?

### Структура бригада → батальйон (когезія)
- [x] Бригади складаються з батальйонів як окремих юнітів на карті; бригада сприймається як єдине ціле
- [x] Когезія: якщо батальйони далеко від HQ або один від одного — readiness знижується
- [x] HQ бригади — статичний, не переміщується (операція коротка)
- [x] HQ відображається на карті як окремий маркер (шеврон + пунктирне коло)
- [x] Клік на HQ → меню директив для бригади
- [x] Директива бригади: Наступ / Утримувати / Відпочинок (активна поки не змінена)
- [x] Зміна директиви через клік на HQ → вибір нової; меню не закривається після вибору
- [ ] Директива впливає на всі роти бригади в реальному часі (логіка не реалізована, тільки UI)
- [ ] Когезія: роти далеко від HQ поступово втрачають readiness і morale (логіка не реалізована)
- [x] Батальйон — логічний контейнер без фізичного HQ на карті; групує роти для когезії і readiness
- [ ] Скільки батальйонів у бригаді? 2–3 чи більше?
- [ ] Когезія впливає тільки на readiness, чи також на доступні тактики і туман війни?

## Реалізація

### Фаза 1 — Карта ✓
- [x] Ініціалізація проєкту: Vite + React
- [x] Встановити залежності: mapbox-gl, react-map-gl
- [x] Вивести карту центровану на зону операції (Балаклія / Куп'янськ / Ізюм)
- [x] Налаштувати bounds, zoom, стиль карти (satellite)
- [x] Маска затемнення за межами зони операції
- [x] Flat-top гексова сітка поверх карти

### Фаза 2 — Юніти на карті ✓
- [x] Визначити модель даних юніта (Brigade → Battalion → Company)
- [x] Розмістити стартові позиції (80, 95, 25 бригади)
- [x] Відрендерити юніти на гексах (NATO символи + шеврони бригад)
- [x] Вибір юніта лівим кліком; скасування вибору — клік на карту
- [x] Переміщення правим кліком на гекс
- [x] Анімоване переміщення між гексами (lerp)
- [x] Zustand store
- [x] TypeScript (міграція з JS)

### Фаза 3 — Панель юніта і штаби ✓
- [x] Бокова панель при виборі роти (назва, бригада, тип, readiness, morale, visionRadius)
- [x] Відображення шеврону бригади в панелі
- [x] Параметр бойового духу (Morale: High / Steady / Shaken / Panic)

- [x] Меню директив при кліку на штаб (Наступ / Утримувати / Відпочинок)

### Фаза 4 — Туман війни
- [ ] Карта затемнена скрізь крім радіусу видимості рот
- [ ] Кожен тип роти має свій visionRadius (UAV=6, Recon=4, Special=3, інші=2)
- [ ] Видимість оновлюється при переміщенні юнітів

### Фаза 5 — Противник (РФ)
- [ ] Скриптовані юніти РФ в обороні
- [ ] Патерни поведінки: hold / withdraw / counterattack
- [ ] Юніти РФ видимі тільки в зоні видимості гравця

### Фаза 6 — Ігровий час
- [ ] Реальний час з паузою
- [ ] Кнопки: пауза / нормальна швидкість / прискорена
- [ ] Таймер операції (6 вересня 2022, день 1)

### Фаза 7 — Ігрова логіка
- [ ] Директива Наступ: +атака, readiness знижується швидше
- [ ] Директива Утримувати: +оборона, заборона переміщення
- [ ] Директива Відпочинок: поступове відновлення readiness і morale
- [ ] Когезія: розрахунок відстані рота→HQ, штраф при перевищенні радіусу
- [ ] Бойовий контакт при зіткненні рот (вирішення поки відкрите питання)

## Рішення
- Flat-top hex grid замість H3 — H3 дає ротацію через ікосаедральну проєкцію, некрасиво виглядає на карті
- Директиви — постійні накази без cooldown; меню залишається відкритим після вибору
- Batallion — логічний контейнер, не має позиції на карті
- HQ бригади — статичний маркер (операція займає 4 дні, штаби не переміщуються)
