# Wargame — План розробки

## Відкриті питання

- [x] **Бойовий контакт** — автоматичний розрахунок кожен тік
- [x] **Тригер бою** — зона контролю (ZoC): юніт у ZoC ворога зупиняється і вступає в бій
- [x] **Модель бою** — авто-розрахунок; концентрація сил знижує вхідний дамаг атакованого
- [x] **Параметр strength** — новий параметр 0–100%, витрачається в бою; readiness і morale — окремі стани
- [x] **Поразка роти** — два сценарії:
  - `strength` → 0: рота знищена, видаляється з мапи
  - `readiness === Exhausted` або `morale === Panic`: рота автоматично відступає до HQ, ігнорує накази гравця
- [x] **Артилерія в бою** — два режими:
  - Режим очікування (авто): автоматично атакує найближчого ворога в зоні ураження
  - Ручне призначення цілі: гравець вибирає артилерію → клік на гекс → пріоритетна ціль
  - Cooldown між пострілами
- [x] **Послідовність реалізації** — гібрид: 2–3 статичні роти РФ для тестування → повна бойова логіка з чистою архітектурою → AI ворога пізніше
- [ ] **Дезінформаційна механіка** — гравець натискає кнопку і витрачає ресурс, чи тонше — через реальні переміщення хибних юнітів?
- [ ] **Туман війни — візуалізація** — як гравець бачить "застарілу" розвідку? Прозорість? Поступове зникнення позицій?
- [x] **Режим гри** — реальний час з паузою і контролем швидкості
- [ ] **Підкріплення** — по ходу гри чи всі юніти з початку?
- [ ] **Туман війни** — видимість на рівні рот чи батальйонів?

### Юніти гравця
- [x] Ієрархія: Бригада (HQ) → Батальйон → Роти (пряме керування гравця)
- [x] Типи рот: Assault, Line, Recon, UAV, Special, Tank, Artillery
- [x] Спецура — невидима у тумані війни
- [ ] Додаткові типи: AntiTank (ПТРК), Engineer, Mortar, StrikeUAV

### Зона контролю (ZoC)
- [ ] За замовчуванням ZoC = 1 гекс для всіх рот
- [ ] Tank: ZoC = 2
- [ ] Line: ZoC = 2; може "окопатись" → ZoC = 4, але не може рухатись
- [ ] Artillery, UAV: ZoC = 0 (не блокують рух)
- [ ] Окопування: 4 ігрові години, вихід з окопів також займає час
- [ ] Окоп дає бонус до захисту (зниження вхідного дамагу)
- [ ] Захисний бонус стакається з ландшафтом (Forest / Urban + окоп = подвійний захист)

### Ролі і баланс юнітів

**Штурмова рота** — кулак прориву
- [ ] Висока атака, середній захист
- [ ] Readiness витрачається швидше ніж у інших

**Лінійна піхота** — оборонна стіна
- [ ] Середня атака, висока стійкість у захисті
- [ ] Окоп: ZoC=4, найкраща оборонна одиниця
- [ ] Окоп: ×0.3 вхідного дамагу від артилерії (−70%), ×0.5 від танків (−50%)
- [ ] Окоп: не може відступати — залишається на позиції навіть при Panic
- [ ] Артилерія по окопаній піхоті: strength захищена, але morale знижується кожен тік під обстрілом
- [ ] Panic в окопі: великий штраф до атаки і захисту, але рота не тікає (дезорганізована оборона)

**Танкова рота** — прорив і переслідування
- [ ] Висока атака по піхоті на відкритому полі (×1.5 дамаг на Open)
- [ ] Штраф у місті і лісі (×0.5 атака на Urban/Forest)
- [ ] Може рухатись навіть у бою (прорив, не обхід)
- [ ] Вразлива до артилерії (×1.5 вхідного дамагу від Artillery)

**Розвідка** — очі армії
- [ ] Швидкість +50%, мала бойова сила
- [ ] Виявляє ССО ворога в радіусі 1 гекс

**ССО** — хірург
- [ ] Невидимий, прорив крізь ZoC, ×3 атака 1х1
- [ ] Знищує артилерію за кілька тіків

**Артилерія** — важіль
- [ ] Вогнева підтримка з відстані, ZoC=0
- [ ] Авто-ціль або ручна ціль по правому кліку

### Спеціальні механіки юнітів
- [ ] **Recon**: швидкість руху +50%
- [ ] **SSO**: невидимі ворогу завжди, крім випадків — у бою або впритул (відстань ≤ 1) до ворога
- [ ] **SSO**: ×3 дамаг проти 1 цілі; ×1.5 проти 2 цілей; ×0.75 проти 3+ (штраф за концентрацію вогню)
- [ ] **SSO**: знищує артилерію за кілька тіків (великий множник дамагу по Artillery)
- [ ] **SSO**: може рухатись крізь ZoC ворога без зупинки
- [ ] **Tank**: ×1.5 атака на Open; ×0.5 атака на Urban/Forest; рухається у бою; ×1.5 вхідного дамагу від Artillery
- [ ] **Assault**: вища атака, readiness витрачається швидше
- [ ] **Відступ під вогнем**: піхота у бою, що отримала наказ відступу → штраф −50% швидкість, +50% вхідний дамаг поки не вийде з ZoC

### Структура бригада → батальйон (когезія)
- [x] HQ бригади — статичний маркер (шеврон + пунктирне коло)
- [x] Клік на HQ → меню директив (Наступ / Утримувати / Відпочинок)
- [x] Батальйон — логічний контейнер без позиції на карті
- [ ] Директива впливає на роти в реальному часі (тільки UI, логіки нема)
- [ ] Когезія: роти далеко від HQ поступово втрачають readiness і morale

## Рішення
- Flat-top hex grid замість H3 — H3 дає ротацію через ікосаедральну проєкцію
- Директиви — постійні накази без cooldown
- Battalion — логічний контейнер, не має позиції на карті
- HQ бригади — статичний маркер (операція 4 дні)
- Terrain — Overpass API замість queryRenderedFeatures (не бачить ліс на zoom 9)
- mapConfig.ts — єдине місце для гео-констант; перенос на інший сектор = 1 файл
