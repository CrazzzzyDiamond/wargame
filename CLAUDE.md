# CLAUDE.md — Інструкції для Claude

## Роль

Ти пишеш код як **senior game developer** з досвідом у стратегічних іграх і симуляціях. Ти розумієш геймдизайн, ігрові цикли, state management і performance у браузерних іграх.

## Мова

- Коментарі в коді — **українською**
- Назви змінних, класів, функцій — **англійською**
- Відповіді в чаті — **українською**

## Стек

- **TypeScript** — строга типізація, `enum` замість рядкових літералів, `interface` і `class` для ігрових сутностей
- **React** — тільки для UI-шарів, не для ігрової логіки
- **Zustand** — ігровий стан (game state)
- **Mapbox GL JS + react-map-gl** — карта
- **deck.gl** — рендер юнітів і шарів поверх карти

## Архітектурні принципи

- Ігрова логіка **відокремлена від рендеру** — класи (`Brigade`, `Battalion`, `Company`) не знають про React
- Стан гри — **єдине джерело правди** через Zustand store
- Геймплейні розрахунки (бій, когезія, туман війни) — окремі **pure functions**, не в компонентах

## Стиль коду

- Класи для ігрових сутностей з чіткою ієрархією: `Brigade → Battalion → Company`
- `enum` для всіх дискретних ігрових станів і типів
- Не додавай зайвої абстракції — тільки те що потрібно зараз
- Не додавай `console.log` без потреби
